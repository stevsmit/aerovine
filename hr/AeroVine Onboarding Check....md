## **2\. IT & Hardware Setup (Day 1/Week 1\)**

**Responsible:** IT or Software Department Lead

| Status | Task | Details |
| :---- | :---- | :---- |
| $\\Box$ | **Workstation Provided** | Assign and set up the workstation (Desk, Monitor, Ergonomic accessories). |
| $\\Box$ | **Laptop/Computer Issued** | Issue the primary machine (**MacBook Pro** or Linux Workstation for Engineers). |
| $\\Box$ | **Required Software Installed** | Ensure core applications are installed: **VS Code**, **LibreOffice**, Slack, Video Conferencing client. |

| $\\Box$ | **Network Credentials** | Provide Wi-Fi credentials and credentials for internal servers/network drives. |

## **3\. Software/Engineering & IP Access (Week 1 Focus)**

**Responsible:** Manager and Senior Engineer/Team Lead

| Status | Task | Details |
| :---- | :---- | :---- |
| $\\Box$ | **Git & GitHub Access** | Create account, grant access to the main **AeroVine** code repositories (read and write). |
| $\\Box$ | **Project Management Access** | Grant access to the project tracking system (e.g., **Jira** or Trello) and assign initial tickets. |
| $\\Box$ | **Cloud Credentials** | Issue credentials for necessary cloud platforms (e.g., AWS S3 bucket for training data, Azure DevOps, CI/CD pipeline access). |
| $\\Box$ | **Code Base Orientation** | Walk through the structure of the main code repositories, including the core flight control software and the AI/Computer Vision models. |
| $\\Box$ | **First Commit Training** | Guide the new hire through cloning the repository, making a small, non-critical change (e.g., to the README), committing, and pushing the change. |
| $\\Box$ | **Local Dev Environment Setup** | Ensure the employee can successfully run the **local drone simulation** or a core module of the AI software on their machine. |
| $\\Box$ | **Drone Lab Access** | If role requires, grant access to the physical drone testing lab and explain safety zones/protocols. |
